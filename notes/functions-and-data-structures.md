# Functions and Data Structures

## Module 2 Notes

---

## 🔹 Functions

### What it is
A reusable block of code that performs a specific task.

### Backend Meaning
- Represents **logic units inside a server**
- Used to handle:
  - validation
  - processing
  - response formatting

### Flow
Input → Function → Output

---

## 🔹 Variable Scope

### What it is
Where a variable can be accessed in a program.

### Types
- Local → inside function
- Global → outside function

### Backend Meaning
- Controls **data visibility**
- Prevents unintended data modification
- Keeps logic isolated

---

## 🔹 Function and Variable Scope

### Key Idea
Functions create their own scope.

### Backend Thinking
- Each function acts like a **mini-service**
- Data inside stays controlled and predictable

---

## 🔹 What Are Data Structures?

### What it is
Ways to organize and store data.

### Backend Meaning
- Determines how data is:
  - stored
  - accessed
  - processed

---

## 🔹 Lists

### What it is
Ordered collection of items.

### Example Use
- storing multiple values

### Backend Use
- handling collections (e.g. user data, records)

---

## 🔹 Tuples

### What it is
Ordered, **immutable** collection.

### Backend Use
- fixed data (coordinates, configs)

---

## 🔹 Sets

### What it is
Unordered collection of **unique items**

### Backend Use
- removing duplicates
- membership checks

---

## 🔹 Quick Backend Mapping

| Concept | Backend Role |
|--------|--------|
| Function | Logic handler |
| Scope | Data control |
| List | Collection of records |
| Tuple | Fixed data |
| Set | Unique filtering |

---

## 🔹 Key Takeaway

- Functions = reusable backend logic  
- Scope = control data flow  
- Data structures = how backend stores and processes data  

---

## 🔹 Personal Notes

- Functions help reduce repetition  
- Scope prevents bugs from shared variables  
- Data structures impact performance and clarity

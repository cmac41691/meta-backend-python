# Procedural Programming

## What is Procedural Programming

Procedural programming is writing code as a sequence of steps executed in order.

- Top → bottom execution
- Each step runs after the previous one
- No skipping unless controlled by conditions or loops

---

## Algorithms

An algorithm is a set of steps used to solve a problem.

Example:
- Input → Process → Output

Backend example:
- Receive request
- Validate data
- Process logic
- Return response

---

## Writing Algorithms

Breaking a problem into clear steps before coding.

### Example (Coffee)

1. Boil water
2. Add coffee grounds
3. Pour water
4. Wait
5. Serve

---

## Algorithmic Complexity

Measures how efficient an algorithm is.

Focus:

Time (how fast)

## Big-O Notation

Used to describe performance as input grows.

Examples:

O(1) → constant
O(n) → linear
O(n²) → quadratic
